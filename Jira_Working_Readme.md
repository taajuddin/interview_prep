## ðŸš€ How Jira Works (Short & Practical)

### **1. Project**

A project contains all tasks (issues) related to your product.

### **2. Issue Types**

-   **Epic** -- Large module
-   **Story** -- Feature request
-   **Task** -- Work item
-   **Bug** -- Issue to fix

### **3. Workflow**

Issues move through stages: - To Do
- In Progress
- In Review
- Testing
- Done

### **4. Boards**

-   **Scrum** -- Sprint-based
-   **Kanban** -- Continuous flow

### **5. Sprint Cycle**

1.  Sprint Planning
2.  Development
3.  Daily Standup
4.  Review
5.  Retrospective

### **6. Backlog**

All upcoming tasks stored and prioritized.

### **7. Integrations**

GitHub, Bitbucket, CI/CD, Slack, Confluence, etc.
